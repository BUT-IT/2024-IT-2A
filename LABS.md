#  Лабораторная работа №1

## Тема: Верстка макета из Figma — HTML5 + CSS3 + SASS

###  Цель работы

Развить навыки современной веб-верстки: работа с HTML5-семантикой, CSS3-модулями, адаптивной раскладкой с использованием CSS Grid и Flexbox, а также препроцессора SASS. На основе готового макета в Figma создать рабочую страницу, максимально приближенную к дизайну.

---

###  Исходный макет

Используйте ссылку на файл дизайна в Figma:
[Макет для Лабораторной №1](https://www.figma.com/design/oppkU5rbQNg0V7EtFUnQJL/lab_1?node-id=0-1&p=f&t=Mh2X4JusX95y0thD-0)
Внимательно ознакомьтесь с расположением блоков, типографикой, отступами, цветовыми акцентами и адаптивностью.

---

### Технологии и требования к реализации

* **HTML5**: использовать семантические теги (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>` и т.д.).
* **CSS3**: использовать современные возможности:

  * Flexbox для линейных раскладок (горизонтальных/вертикальных элементов).
  * CSS Grid для комплексных сеток или макета со столбцами и строками.
  * Адаптивность: страница должна корректно отображаться на разных ширинах (например, на десктопе и планшете).
* **SASS** (или SCSS-синтаксис): использовать препроцессор для:

  * вложенности селекторов;
  * переменных (`$color-primary`, `$font-size-base` и т.д.);
  * миксинов или функций (если требуется);
  * импорта частей (`_header.scss`, `_footer.scss`, `_components.scss`).
* Стиль должен максимально соответствовать макету: шрифты, цвета, отступы, визуальные эффекты (например, hover-состояния), изображения (если есть в макете).
* Код должен быть чистым и структурированным: разбит на логические секции, комментарии по необходимости, использование BEM-или эквивалентной методологии по стилям.

---

### Что проверить в результате

* Верстка корректно отображается в браузерах (Chrome, Firefox) на **ширинах**: десктоп (например, >= 1024 px) и планшет/мобильный (например, <= 768 px).
* Нет горизонтальной прокрутки на типовой ширине браузера.
* Стили адаптируются: меню, сетка, картинки корректно «съезжают» на меньших экранах.
* Семантическая структура HTML — логично используются теги, нет избыточных `div`-обёрток.
* SASS-структура: разбито на модули/части, применены переменные и вложенность, минимальная дубликация кода.
* Макет визуально соответствует Figma-дизайну: одинаковые шрифты/цвета/отступы (в разумных пределах браузерного отображения).
* Код читабелен: отступы, инденты, комментарии, именование классов соответствующее.

* (опционально) изображения, используемые на странице;

* краткий отчёт либо комментарий к коду, где указаны: какие технологии были применены (Grid, Flexbox, SASS), с какими трудностями столкнулись и как решили.

---

### Рекомендации и полезные советы

* Сначала создайте **каркас HTML** без стилей, разместите все блоки.
* Затем напишите базовые стили SASS: переменные цвета и типографики.
* Реализуйте раскладку страницы через CSS Grid или Flexbox (как подходит макету).
* После — адаптивность: используйте медиа-запросы (`@media`) или гибкие единицы (`rem`, `fr`, `%`).
* Минимизируйте повторение кода: используйте миксины/платформенные классы.
* Проверяйте промежуточный результат в браузере, корректируйте стили под макет в Figma.
* Используйте инспектор браузера (DevTools) чтобы сверять отступы и размеры элементов.
* Если изображения большие — оптимизируйте или масштабируйте, чтобы не тормозила загрузка.

# Лабораторная работа №2

## Тема: Разработка веб-приложения «Менеджер покупок» на JavaScript

### Цель работы

Закрепить навыки работы с основными элементами JavaScript:

* переменные и типы данных;
* операторы, условия и циклы;
* объекты, массивы и функции;
* локальные и глобальные переменные;
* объявление, вызов, анонимные и колбэк-функции;
* работа с DOM API и использование jQuery.

---

## Описание задачи

Разработать веб-приложение **«Менеджер покупок»**, которое позволяет пользователю вести список покупок с возможностью добавления, пометки как «куплено», удаления и фильтрации элементов.

---

##  Основные требования

### 1. Функционал приложения

* Пользователь может **добавлять позиции** с указанием:

  * названия (string),
  * цены (number),
  * количества (integer);
* Список хранится в **массиве объектов**;
* Реализован **расчёт общей стоимости корзины** (сумма всех товаров);
* Добавлена возможность:

  * **пометить** позицию как «куплено» (переключение состояния — *toggle*);
  * **удалить** позицию полностью;
* Реализован **фильтр списка**:

  * показывать: все / купленные / некупленные (через условие и циклы);
* При клике на кнопку **«Куплено»** элемент должен визуально отличаться (например, зачёркнут текстом или цветом);
* Общая стоимость корзины отображается **отдельно** и обновляется при каждом изменении.

---

### 2. Технические детали реализации

* Интерфейс состоит из формы с полями:

  * **Название**
  * **Цена**
  * **Количество**
  * Кнопка **«Добавить»**
* Каждая позиция отображается в виде блока (или строки таблицы) с:

  * названием;
  * ценой;
  * количеством;
  * суммой строки *(цена × количество)*;
  * кнопкой **«Куплено»** (изменяет состояние);
  * кнопкой **«Удалить»** (удаляет позицию);
* Применяется **DOM API** для динамического рендера списка;
* Используются **локальные и глобальные переменные** (в отчёте указать, какие и зачем);
* Применяются разные типы функций:

  * объявленные (`function имя() {}`),
  * функциональные выражения (`const f = function() {}`),
  * стрелочные (`() => {}`),
  * анонимные функции (например, в `addEventListener`);
* Добавлен **пример колбэк-функции**, например:

  ```js
  function updateAndThen(callback) {
    // обновляем данные
    if (typeof callback === "function") callback();
  }
  ```
* Реализована хотя бы **одна операция с использованием jQuery**:

  * например, плавное появление новой строки (`.slideDown()`),
  * или подсветка добавленного элемента (`.css()` / `.fadeIn()`).

---

### 3. DOM и визуальные эффекты

* При добавлении/удалении элемента интерфейс **автоматически обновляется**;
* При отметке «куплено» строка **меняет стиль** (например, зачёркивание или зелёный фон);
* При использовании jQuery можно добавить **анимацию**:

  ```js
  $(newRow).hide().slideDown(300);
  ```
